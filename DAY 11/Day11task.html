Interview Questions on HTML

1.What is HTML?

HTML (HyperText Markup Language) is the standard language used to create and structure content on the web.

2.Why do we use HTML?

We use HTML (HyperText Markup Language) because it provides the structure and foundation for webpages.

3.What is Doctype in HTML?

The !DOCTYPE declaration in HTML (commonly referred to as "doctype") is a statement that defines the version of HTML that the webpage is written in. It tells the browser how to interpret and render the HTML document, ensuring that the content is displayed correctly and consistently across different browsers.

4.What is root element of HTML?

The root element of an HTML document is the html tag. It is the topmost, or "root," element that contains all other HTML elements within a webpage. Every HTML document starts with the html starting tag and ends with the /html closing tag.

5.what is a tag?

A tag in HTML is a piece of code used to define elements and structure within an HTML document. Tags are the basic building blocks of HTML, and they tell the browser how to display content like text, images, links, forms, etc.

6.Give me a few examples of self-closing tags?

Few examples of self-closing tags are image tag,break tag,hr tag,input tag,meta tag & link tag.

7.What is an element?

An element in HTML is a complete structure that consists of an opening tag, content, and a closing tag (for paired elements), or just a single self-closing tag (for void elements).

8.How many types of elements are there in HTML?

There are mainly two types of elements in HTML.They are:
1.Block elements
2.Inline elements

9.What is the difference between inline vs block elements?

The main difference is that block elements take up the full width and start on a new line, while inline elements take up only as much width as their content needs and remain in the flow of text without breaking to a new line.

10.What is the difference between semantic and non-semantic elements?

Semantic elements clearly describe their meaning and role in the content (e.g., article, header, footer, nav), while non-semantic elements do not provide any meaning about their content (e.g., div, span).

11.What are href vs src attributes?

The href attribute specifies a URL for linking to another page or resource (e.g., in anchor tags), while src specifies the source URL for embedding content like images or scripts (e.g., in image tags).

12.What are the features of HTML5 ?

HTML5 features include new semantic elements (like `header tags`, `article tags`), native multimedia support with `audio tag` and `video tag`, the `canvas` element for 2D graphics, enhanced form controls, geolocation, local storage, web workers for background tasks, and improved support for responsive design and accessibility.

13.What are entities in HTML?

In HTML, entities are special codes that represent reserved characters or symbols that either can't be used directly in the code (like < or &) or are difficult to type (like accented characters). Entities begin with an ampersand (&) and end with a semicolon (;).

14.What is div vs span?

In HTML, the div and span tags are used for structuring and styling content. div tag creates block-level containers for grouping elements, while span tag creates inline containers for styling specific portions of text or elements within a block-level container.

17.Mention a few attributes for img tags?

Here are a few common attributes for the img tag in HTML:
1.src (source) - Specifies the path to the image file.
2.alt (Alternative text) - Provides alternative text for the image if it cannot be displayed.
3.width - Defines the width of the image (in pixels or percentages).
4.height - Defines the height of the image (in pixels or percentages).
5.title - Adds a tooltip that appears when the user hovers over the image.
6.class - Allows you to assign a CSS class to the image for styling purposes.

18.What is thead on the table?

The thead element in an HTML table is used to group the header content of a table. It is typically placed at the top of the table and contains the column headings, which are usually defined within th (table header) tags. The thead tag helps with organizing and styling table headers, making it easier for browsers and screen readers to understand and process the structure of the table.

19.What is tr?

In HTML, the tr tag stands for "table row." It is used to define a row within a table. Each tr tag typically contains one or more td (table data) or th (table header) elements, which represent the cells within the row.

20.What is th?

In HTML, the th tag stands for "table header." It is used to define header cells in a table. The content inside a th element is typically bold and centered by default, and it indicates that the cell is a header for either a row or a column in the table.

21.What is td?

In HTML, the td tag stands for "table data." It is used to define a cell in a table that contains data (as opposed to a header cell, which is defined using th tag). Each td element represents a single cell within a row (tr) of a table.

22.Can I use th in tbody?

Yes, you can use the th tag inside a tbody element in HTML. Although the th tag is commonly used in the thead section to define headers for the columns, it can also be used in the tbody section to define headers for rows.This is particularly useful when you want to create a table where each row has a descriptive label or header, typically displayed in the first column.

Meta Tags:
Meta tags are HTML tags that provide metadata (data about data) about a web page. They are placed inside the head section of an HTML document and are not visible on the webpage itself. Meta tags help browsers, search engines, and other web services understand information about the page, such as its description, keywords, author, and instructions on how the page should be displayed.
SEO:
Search engine optimization (SEO) is the process of improving the quality and quantity of website traffic to a website or a web page from search engines.SEO targets unpaid traffic (known as "natural" or "organic" results) rather than direct traffic or paid traffic. Unpaid traffic may originate from different kinds of searches, including image search, video search, academic search,news search, and industry-specific vertical search engines.

25.What is an iframe and an anchor ?

An iframe, or inline frame, is an HTML element used to embed another HTML document inside the current web page. It allows you to display a separate webpage, or other content like videos, maps, or documents, within a defined area of the parent page.Whereas,an a tag, or anchor tag, is used to create a hyperlink that links one webpage to another or navigates to a different section within the same page. It is one of the most fundamental HTML elements for navigation.

26.What is git vs github?

Git and GitHub are related but serve different purposes in software development. Here's a breakdown of the differences:
1. Git:
Git is a distributed version control system (DVCS) that allows developers to track changes to code, collaborate with others, and manage different versions of a project efficiently. It operates locally on your computer and does not require an internet connection after installation.
2. GitHub:
GitHub is a web-based platform built around Git. It provides a cloud-based hosting service for Git repositories, making it easier for teams to collaborate, share, and manage projects. While Git is a tool for version control, GitHub adds several additional features that facilitate collaboration, code sharing, and project management.

27.What is the workflow of github?

The GitHub workflow generally follows these steps:
1. Set Up a Repository: Create a new repository on GitHub and clone it to your local machine.
2. Create a Branch: Develop features or fix bugs in a separate branch from the main branch.
3. Make Changes: Edit files locally to implement your changes.
4. Stage Changes: Select the modified files you want to include in the next commit.
5. Commit Changes: Save your changes with a descriptive message.
6. Push Changes: Upload your branch and changes to GitHub.
7. Create a Pull Request: Propose merging your changes into the main branch for review.
8. Review and Merge: Collaborators review the pull request, provide feedback, and merge it if approved.
9. Update Local Repository: Synchronize your local repository with the main branch to keep it up to date.
This workflow supports collaborative development and version control.

29.How can you stage all files at a time?

30.What is the git push command and use of it?

The git push command is used to upload local repository content to a remote repository, such as GitHub, GitLab, or Bitbucket. It transfers commits from your local branch to the corresponding branch in the remote repository, making your changes available to others.

31.What is git commit command usage?

The git commit command is used to save changes to the local repository in Git. It creates a snapshot of the current state of the tracked files and includes a descriptive message about the changes made. This is a crucial step in version control, as it allows you to keep a history of your project’s development.

32.What is the use of the git init command?

The git init command is used to create a new Git repository in a specified directory. This command initializes a new local repository, allowing you to start tracking your project files with Git.

33.What are formatting tags?

Formatting tags in HTML are used to alter the appearance of text within a web page. These tags are primarily used to apply various styles to text, making it bold, italic, underlined, or styled in other ways to improve readability and presentation.

34.What are input tag attributes?

The input tag in HTML is used to create interactive controls in a web form that users can use to enter data. The behavior and appearance of an input element can be customized using various attributes. Here are some common attributes for the input tag: type,name,value,id,placeholder,required,disabled,readonly,maxlength,min and max,pattern etc.

35.What are form attributes?

Form attributes in HTML are used to configure the behavior and properties of a <.form> element. These attributes help define how the form operates, where data is sent upon submission, and how the form interacts with the user. Here are some common attributes associated with the <.form> tag: action,name,target etc.

36.What is a marquee tag?

The <.marquee> tag in HTML is used to create scrolling or moving text or images across a web page. It allows developers to display content in a visually dynamic manner, where the text or image moves horizontally or vertically.

37.What are the marquee tag attributes?

The <.marquee> tag has several attributes that control its behavior, appearance, and movement of the scrolling text or images. Here are the commonly used attributes of the <.marquee> tag: direction,scrollamount,behavior,width,height,bgcolor etc.

38.What are multimedia tags?

Multimedia tags in HTML are used to embed audio, video, images, and other media types into a web page. These tags enhance user experience by allowing rich content to be displayed directly within the browser without requiring external applications.img,audio,video and iframe etc. tags are the primary multimedia tags in HTML.

39.Can we use block elements in inline elements?

No, you should not use block elements inside inline elements in HTML. While some browsers may render them, it can lead to unpredictable behavior and layout issues. It's best to nest inline elements within block elements instead.

40.What is the difference between absolute path and relative path?

The difference between absolute and relative paths is:

Absolute Path:
A complete address to a file or resource, starting from the root directory (e.g., /home/user/documents/file.txt or https://www.example.com/images/photo.jpg). It remains the same regardless of the current directory.

Relative Path:
A partial address that specifies the location of a file or resource in relation to the current directory (e.g., documents/file.txt or images/photo.jpg). It can change based on the current directory.

41.What are summary and details tags?

The <.summary> and <.details> tags in HTML are used to create a collapsible section of content that can be expanded or collapsed by the user. This feature is useful for providing additional information or hiding complex content until it's needed.
The <.summary> tag is used as a summary or heading for the content within the <.details> element. It acts as the clickable part that controls the visibility of the details.
The <.details> tag is used to create a disclosure widget from which the user can obtain additional information or controls.

42.What tag is helpful to give the reviews?

The <.blockquote> tag is commonly used to give reviews or citations in HTML. It is designed to represent a section of content that is quoted from another source, making it ideal for displaying reviews, testimonials, or any other quoted text.

43.What are fieldset and legend tags?

The <.fieldset> and <.legend> tags in HTML are used to group related elements within a form, enhancing both structure and accessibility.
The <.fieldset> tag is used to group related form controls (like input fields, checkboxes, and radio buttons) together, visually and semantically. It helps to organize the form elements, making it clearer for users.
The <.legend> tag is used to provide a caption or title for the content within a <.fieldset>. It describes the group of elements contained in the fieldset, improving the accessibility and usability of forms.
